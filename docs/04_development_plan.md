# AI 주식 모니터링 대시보드 개발 및 검증 계획서 (Development & Verification Plan) - 수기 종목 관리, LIKE 검색, 주기 적용 버튼 및 반응형 개정

이 문서는 별도 폴더의 `config/stocks.json` 파일에서 종목 목록을 수기로 관리할 수 있는 기능, 모바일 반응형 UI 고도화 계획, 유연한 문자열 부분 일치(LIKE) 검색 기능, 그리고 갱신 주기를 선택한 뒤 "적용"하는 버튼과 현재 작동 주기 식별 배지 추가 요건을 종합적으로 반영한 보완 계획서입니다.

## 1. 개발 로드맵 및 태스크 (Milestones)

### Step 1: 프로젝트 환경 설정 및 패키지 설치 [완료]
* Next.js 프로젝트 설정 및 라이브러리(`swr`, `lightweight-charts`, `lucide-react`) 설치 완료.

### Step 2: 수기 종목 관리 파일 추가 및 API Route 개정
* **목적**: 사용자가 직접 종목을 관리할 수 있는 `config/stocks.json` 파일을 추가하고, 서버 사이드 API가 이를 동적으로 참조하여 데이터를 조회하게 합니다.
* **주요 작업**:
  * 별도 설정 디렉토리 내에 **[config/stocks.json](file:///Users/jw/workspace/study/ai/ai-pocketstock/config/stocks.json)** [NEW] 생성 (국내 주식 TOP 10 종목을 기본 탑재하여 기생성 완료)
  * `src/app/api/stocks/route.ts` 에서 `config/stocks.json`을 읽어와서 동적으로 API 쿼리 주소를 생성하도록 개정.

### Step 3: CSS Modules 기반 디자인 시스템 및 다크모드 보완
* **목적**: 다크모드 및 모바일용 유틸리티 클래스를 반영하도록 CSS를 고도화합니다.
* **주요 작업**:
  * `src/app/globals.css` 에 모바일 반응형 및 다크모드용 CSS 변수 보완
  * 스크롤바 감추기 유틸리티 CSS (`.scrollbar-none`) 적용
  * Glassmorphism 카드의 다크모드 테마 스타일 보완 (`.dark .glass-card` 스타일 적용)

### Step 4: UI 컴포넌트 개발 및 상태 관리 (반응형, LIKE 검색, 주기 수동 적용 및 동적 종목 연동)
* **목적**: 모바일용 내비게이션 드로어, 검색 오버레이, 데스크톱용 검색 추천 드롭다운 등을 구현하고 동적 종목 데이터와 LIKE 검색, 수동 적용 갱신 주기 기능을 연동합니다.
* **주요 작업**:
  * **갱신 주기 설정 수동 적용 및 식별 UI 구현**:
    - 헤더에 주기 선택 셀렉터, `[적용]` 버튼, 그리고 현재 작동 중인 주기를 보여주는 텍스트 배지(`(5s)`)를 미니멀하게 설계.
    - 선택 상태(`tempInterval`)와 실제 활성화된 SWR용 주기 상태(`activeInterval`)를 분리하여, 적용 버튼을 클릭했을 때만 SWR 주기가 바뀌어 네트워크 요청 주기가 변하도록 제어.
  * **유연한 LIKE 검색 알고리즘 구현**:
    - 검색창 입력값과 종목명/종목코드 간 대소문자 무시, 공백 무시, 부분 일치(LIKE) 조건 검증 로직 작성.
  * `src/components/Header.tsx`
    - 모바일용 햄버거 메뉴 및 검색 버튼 연동
    - 데스크톱용 검색창 포커스 시 추천 종목 드롭다운 구현 (LIKE 검색 필터링 적용)
    - 미니멀 갱신 주기 적용 제어 영역 구성
  * `src/components/MobileDrawer.tsx` [NEW]: 모바일 네비게이션 드로어
  * `src/components/MobileSearchOverlay.tsx` [NEW]: 모바일 검색창 및 LIKE 검색 필터링 연동 추천 레이어
  * `src/components/StockList.tsx` (종목 선택 상태 및 LIKE 검색 필터링)
  * `src/components/StockChart.tsx` (반응형 차트 너비 및 가로 스크롤)
  * `src/components/StockSummary.tsx` (우측 날씨 요약 및 핵심 지표 카드)
  * `src/app/page.tsx` 에서 반응형 상태 통합 제어 및 SWR 데이터 바인딩

## 2. 검증 및 테스트 계획 (Verification & Test Plan)

### 테스트 시나리오 및 검증 항목

| 테스트 ID | 대상 기능 | 검증 방법 및 기대 결과 |
| :--- | :--- | :--- |
| **TC-01** | 환경 설정 | `npm run build` 및 `npm run dev` 시 정상적으로 로컬 서버가 빌드 및 구동되는지 확인. |
| **TC-02** | 프록시 API | `/api/stocks` 호출 시, `config/stocks.json`을 올바르게 읽어 CORS 오류 없이 실시간 데이터가 JSON 포맷으로 반환되는지 확인. |
| **TC-03** | 테마 토글 | 다크 모드/라이트 모드 버튼 클릭 시 `document.documentElement`에 `dark` 클래스가 정상 토글되며 Glassmorphism 카드 테마가 실시간 변환되는지 확인. |
| **TC-04** | 실시간 폴링 | SWR이 `/api/stocks`에 자동으로 요청을 보내고 UI가 갱신되는지 네트워크 탭을 통해 확인. (기본 5초) |
| **TC-05** | 폴링 주기 적용 버튼 | 헤더에서 주기를 변경한 후 **적용 버튼을 누르기 전에는 주기가 바뀌지 않고, 적용 버튼을 누르는 즉시** 네트워크 탭의 `/api/stocks` 호출 주기가 설정값으로 전환되는지 검증. |
| **TC-06** | 주식 차트 | `lightweight-charts` 차트 캔버스 영역에 선택된 종목의 캔들스틱, 이동평균선(5/10/20/60일선), 거래량 차트가 올바르게 정상 표출되는지 확인. |
| **TC-07** | 종목 선택 변경 | 검색 바 또는 목록에서 다른 종목을 선택했을 때 차트 및 우측 요약 카드의 데이터가 해당 종목으로 즉시 업데이트되는지 확인. |
| **TC-08** | 모바일 드로어 | 모바일 해상도(768px 미만)에서 헤더 햄버거 메뉴 버튼 클릭 시 네비게이션 드로어가 왼쪽에서 슬라이드인 형태로 나타나며, X 닫기 버튼 또는 외부 영역 클릭 시 정상적으로 닫히는지 확인. |
| **TC-09** | 모바일 검색 | 모바일 해상도에서 돋보기 버튼 클릭 시 화면 전체를 덮는 검색 오버레이가 나타나며 검색어 필터링 및 인기 종목 클릭으로 종목을 변경할 수 있는지 확인. |
| **TC-10** | 추천 드롭다운 | 데스크톱 해상도에서 검색창 포커스 시 추천 종목 드롭다운 목록이 열리고, 다른 곳을 클릭하면 닫히는지 확인. |
| **TC-11** | 수기 종목 추가 | **[config/stocks.json](file:///Users/jw/workspace/study/ai/ai-pocketstock/config/stocks.json)** 에 수기로 새 종목(예: `035420`, NAVER)을 추가한 뒤, 서버 리빌드 없이 화면의 종목 리스트와 검색창에서 해당 신규 종목이 동적으로 노출되고 시세가 갱신되는지 확인. |
| **TC-12** | LIKE 검색 기능 | 검색창에 대소문자를 다르게 하거나 일부 공백을 넣어도 유연하게 해당 종목이 자동 추천 및 필터링되는지 검증. |
| **TC-13** | 작동 주기 표출 | UI 헤더 상에 현재 작동 중인 실시간 주기가 시각적 배지(예: `5s` 혹은 `현재: 5초`) 형태로 올바르게 실시간 연동되어 출력되는지 검증. |
